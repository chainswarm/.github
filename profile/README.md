## Architecture

### Design Philosophy

> *"We don't need miners for mere execution. We need miners for breakthroughs."*

Chain Insights separates infrastructure from innovation:

- **Subnet Owner** operates core indexing infrastructure, ensuring a reliable "source of truth" across supported blockchains
- **Miners** compete to provide intelligence improvementsâ€”better algorithms, superior models, validated labels

This separation allows miners to focus entirely on advancing capabilities rather than maintaining servers. It also ensures production stability while enabling rapid innovation.

### Core Indexing Service

The foundation layer provides:

- **Unified Data Model:** All assets converted to USD value, enabling cross-chain analysis without currency-specific complexity
- **Aggregated Money Flows:** Focus on value relationships between addresses rather than individual transactionsâ€”smaller, faster, more analytical
- **Rich Feature Sets:** Pre-calculated ML features per address, volume time series, transaction graph metrics
- **Multi-Chain Support:** Blueprints ready for Substrate, EVM, and UTXO chains

This model is fundamentally different from traditional blockchain indexing. By aggregating flows and focusing on USD value, the dataset is compressed, queries are faster, and analysis centres on what matters: money movement patterns.

### Four Mechanisms of Intelligence

**Analytics Mechanism**
Miners submit optimised code for graph algorithms and pattern detection. Submissions are benchmarked against Chain Synthetics test data. Winners see their code deployed to production.

*Participants: Software engineers, algorithm researchers*
*Output: Fraud detection for mixers, layering networks, peel chains, and many more...*

**Labelling Mechanism**
Miners discover and verify address labels from public sourcesâ€”exchange wallets, sanctioned addresses, known scams, hack-related addresses. Each label requires cryptographic evidence verified against the indexing service.

*Participants: OSINT specialists, data hunters*
*Output: Comprehensive address intelligence database*

**Machine Learning Mechanism**
Miners train risk scoring models using features from the indexing service and labels from the labelling mechanism. Models are evaluated against ground truth data and validated over time for real-world accuracy.

*Participants: Data scientists, ML engineers*
*Output: Real-time risk scores for any address*

**Intelligence Mechanism**
Miners fine-tune large language models for AML, blockchain forensics, and investigative queries. Models power the Chain Insights AI chat interface.

*Participants: AI engineers, LLM specialists*
*Output: Natural language investigation capabilities*

### Chain Synthetics: Objective Benchmarking

A critical challenge in fraud detection is measuring accuracyâ€”there is no public "ground truth" dataset of labelled fraudulent activity. Chain Synthetics solves this by:

- **Injecting synthetic patterns** into real blockchain data (known mixers, layering topologies, and many more...)
- **Blending naturally:** Synthetic patterns are blurred and merged with existing transactions so they appear natural within the transaction graphâ€”this directly impacts algorithm design quality
- **Replaying historical hacks** (recreating famous exploits within test environments)
- **Providing objective metrics** (precision, recall, speedâ€”no subjective judging)

This enables tournament-based competition where the best submissions win based on measurable outcomes.

### Tournament-Based Emissions

Unlike some subnets that reward continuous infrastructure provision, Chain Insights uses tournament-based emissions:

- A miner who improves pattern detection by 5% earns emissions for that innovation
- Once the improvement is absorbed into the baseline, emissions stop
- Miners must innovate again to earn again

This model aligns incentives with progress. Stagnation is not rewarded. Breakthroughs are.
